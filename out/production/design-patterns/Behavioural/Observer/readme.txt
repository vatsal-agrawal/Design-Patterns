The Observer design pattern is modeled on the event-driven programming paradigm.
Just like other design patterns, this pattern lets us define loosely coupled systems.
Leveraging this pattern, we can write maintainable and modular software.
Additionally, we can achieve clear segregation between different actors in an event-driven system
